# Movie Industry SQL Analysis

Project Overview

This project analyzes a movie dataset containing 7,668 films released between 1980 and 2020. The objective is to answer real-world business questions using SQL queries in PostgreSQL.

The dataset includes movie attributes such as title, genre, rating, director, actor, production budget, box-office gross, runtime, and country.

Dataset Details

- Total Movies: 7,668
- Time Period: 1980 – 2020
- Columns: 15
- Genres: 15+
- Countries: 60+

Business Questions Answered

1. Which genres deliver the highest ROI?
2. Which directors generate the most total profit?
3. Do higher IMDb ratings lead to higher box-office revenue?
4. Which movies failed to recover their budget?
5. Which studios dominated box-office revenue by decade?
6. Which actors generate the highest box-office revenue?
7. How has industry revenue changed year over year?
8. Which genres became more popular over time?
9. What runtime range produces the best performance?
10. What is the highest-grossing film for each country?
11. Are movie budgets increasing faster than revenues?
12. Which content rating performs best commercially?
13. Which writer-director collaborations produce the highest ratings?

SQL Skills Demonstrated

- Aggregation and Grouping (SUM, AVG, COUNT)
- Window Functions (LAG, RANK, SUM OVER)
- Conditional Logic using CASE WHEN
- Subqueries and Ranking
- Data Cleaning and NULL Handling
- Business Metric Calculations (ROI, Growth Rate, Market Share)

Tools Used

- PostgreSQL
- SQL
- CSV Dataset

Project Purpose

The project demonstrates how SQL can be used to analyze large datasets and generate business insights similar to those used in the media and entertainment industry.
